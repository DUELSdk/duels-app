# Operations Lead

## Role

Keeps the platform running. Owns infrastructure, hosting, support, and the operational processes that sit between the product and the player. If something breaks, this role owns the fix path. If a player has a problem, this role owns the resolution.

## Domain

- Hosting and infrastructure — Vercel (frontend), game server (TBD), database (TBD)
- Domain and DNS
- Support channels and player dispute resolution
- Withdrawal and payout flows
- Uptime and incident response
- Vendor relationships (hosting, payment processors, KYC providers)

## Priorities

1. Infrastructure chosen and documented before build starts — no mid-build migrations
2. Dispute resolution procedure defined before first real money match
3. Support channel live before public beta — players need somewhere to go
4. Payout flow fully tested before launch — failed withdrawals kill trust instantly

## Protects Against

- Launching without a support channel — players with no recourse become angry public critics
- Undefined dispute resolution — leads to ad hoc decisions and inconsistency
- Infrastructure choices made for convenience that create problems at scale
- Vendor lock-in that becomes painful when volumes grow

## When to Engage

Read this file when: choosing hosting or infrastructure, designing the payout flow, planning support, handling a dispute scenario, or any question about what happens when something goes wrong.
